## User Story

Como **usuário**, quero receber uma notificação no WhatsApp assim que meu conteúdo for publicado, com o link direto de cada post em cada plataforma, para confirmar que tudo deu certo e compartilhar rapidamente.

---

## Acceptance Criteria

**AC1 — Notificação de sucesso total (todas as plataformas ok)**

```
Dado que um post foi publicado com sucesso em todas as plataformas
Quando a publicação é confirmada
Então o bot envia mensagem proativa para o número do usuário:

"🎉 Post publicado com sucesso!

📱 Instagram → instagram.com/p/ABC123
🎵 TikTok → tiktok.com/@user/video/DEF456
💼 LinkedIn → linkedin.com/posts/user_GHI789
▶️ YouTube → youtu.be/JKL012

📊 Quer ver as métricas depois? Acesse: postai.app/dashboard"
```

**AC2 — Notificação de sucesso parcial (alguma plataforma falhou)**

```
Dado que algumas plataformas publicaram e outras falharam
Quando a publicação é processada
Então o bot envia:

"⚠️ Post publicado com pendências:

✅ Instagram → instagram.com/p/ABC123
✅ TikTok → tiktok.com/@user/video/DEF456
❌ LinkedIn → Falhou (token expirado)
❌ YouTube → Falhou (quota excedida)

Reconecte LinkedIn e YouTube: postai.app/settings/social
Ou tente novamente: responda 'retentar {campaign_id}'"
```

**AC3 — Notificação de falha total**

```
Dado que todas as plataformas falharam
Quando a publicação é processada
Então o bot envia:

"❌ Não foi possível publicar. Erro: {motivo resumido}

Tente novamente: responda 'retentar {campaign_id}'
Ou acesse: postai.app/dashboard para ver detalhes"
```

**AC4 — Notificação de agendamento executado**

```
Dado que um post agendado foi executado no horário programado
Quando o job BullMQ processa a publicação
Então o bot envia a notificação da AC1 ou AC2 (conforme resultado)
E inclui: "📅 Post agendado para este horário foi publicado automaticamente."
```

**AC5 — Notificação para agência com identificação do cliente**

```
Dado que Marina publicou como cliente "Clínica Beleza Total"
Quando a publicação é confirmada
Então a notificação inclui: "👤 Cliente: Clínica Beleza Total"
E os links são das contas da clínica
```

**AC6 — Link real do post por plataforma**

```
Dado que a publicação foi concluída
Quando o sistema constrói a notificação
Então busca o post_id retornado pela API de cada plataforma
E monta o link no formato correto de cada plataforma:
- Instagram: https://www.instagram.com/p/{media_id}/
- TikTok: https://www.tiktok.com/@{username}/video/{video_id}
- LinkedIn: https://www.linkedin.com/posts/{post_id}
- YouTube: https://youtu.be/{video_id}

Se a plataforma não retornar link (ex: LinkedIn API): omitir o link e informar "✅ Publicado"
```

**AC7 — Retentar plataformas que falharam**

```
Dado que o usuário respondeu "retentar {campaign_id}" após falha parcial
Quando o sistema processa o comando
Então tenta publicar APENAS nas plataformas que falharam (não re-publica as que já tiveram sucesso)
E envia nova notificação com resultado da retentativa
```

## Notas Técnicas

- Notificação proativa: usar a mesma WhatsApp API para enviar mensagem sem aguardar resposta do usuário
- Guardar post_id por plataforma em `campaign_results` (JSONB ou tabela separada)
- Para posts agendados: o job processor deve chamar o mesmo notification service da publicação imediata
- Rate limiting: WhatsApp Business API limita mensagens proativas — verificar template approval para notificações fora de sessão de 24h (pode exigir HSM template aprovado pelo Meta)

## Dependências

- Story 4.2 (agendamento) — job processor deve chamar notification service
