## User Story

Como **gestora de agência**, quero cadastrar meus clientes no dashboard e conectar as redes sociais de cada um separadamente, para gerenciar todos através do PostAI sem misturar credenciais.

---

## Acceptance Criteria

**AC1 — Acesso ao painel de clientes**

```
Dado que Marina tem plano 'agency'
Quando acessa postai.app/agency/clients
Então vê lista de clientes com: nome, plataformas conectadas, data de criação
E botão "Novo Cliente"
```

**AC2 — Criar novo cliente**

```
Dado que Marina clica em "Novo Cliente"
Quando preenche: Nome do cliente, segmento (opcional), nota interna (opcional)
E confirma
Então o sistema cria agency_client vinculado à conta de Marina
E redireciona para a página do cliente para conectar redes sociais
```

**AC3 — Conectar redes sociais do cliente**

```
Dado que Marina está na página /agency/clients/{client_id}/social
Quando clica em "Conectar Instagram do cliente"
E o cliente (ou Marina com permissão) conclui o fluxo OAuth
Então o token é salvo vinculado ao client_id (não ao user_id de Marina)
```

**AC4 — Limite por plano**

```
Dado que Marina tem plano Agency (limite: 10 clientes)
Quando tenta criar o 11º cliente
Então exibe: "Limite do plano atingido. Faça upgrade para o plano Agency Pro."
```

**AC5 — Deletar cliente**

```
Dado que Marina quer remover um cliente
Quando confirma a exclusão
Então o sistema revoga os tokens de todas as plataformas do cliente
E remove agency_client e api_tokens vinculados
E NÃO remove as campanhas históricas (arquivadas)
```

---

## Notas Técnicas

- Limites por plano: Starter=0, Creator=0, Agency=10, Agency Pro=ilimitado
- Verificar plano antes de criar cliente (middleware de autorização)
- Dados do cliente são da agência, não do usuário final — LGPD: agência é responsável pelos dados

---

## Dependências

- Story 3.2 (OAuth) para poder conectar as redes do cliente
- Schema `agency_clients` criado
