## User Story

Como **criador de conteúdo**, quero agendar um post para uma data e hora específicas via WhatsApp, para preparar conteúdo com antecedência e publicar no melhor horário sem precisar estar online.

---

## Acceptance Criteria

**AC1 — Iniciar fluxo de agendamento**

```
Dado que o usuário está no menu principal
Quando digita "2" ou "agendar"
Então o estado muda para 'waiting_for_scheduled_copy'
E o bot responde: "📅 Vamos agendar um post! Qual é a legenda?"
```

**AC2 — Coleta de data e hora**

```
Dado que o usuário enviou copy + vídeo no fluxo de agendamento
Quando o bot solicita a data
Então responde: "📅 Para quando? (ex: amanhã 18h, 10/04 14:30 ou 'próxima segunda 9h')"

Dado que o usuário enviou uma data válida no futuro
Quando o bot parseia a data (linguagem natural ou formato explícito)
Então confirma: "Agendar para terça-feira, 09/04 às 18:00?"
```

**AC3 — Data inválida ou no passado**

```
Dado que o usuário enviou uma data no passado ou inválida
Quando o bot tenta parsear
Então responde: "⚠️ Data inválida ou no passado. Informe uma data futura. Ex: amanhã 18h"
E mantém o estado aguardando a data
```

**AC4 — Confirmação de agendamento**

```
Dado que o usuário confirmou data e conteúdo
Quando digita "confirmar"
Então o sistema salva a campanha com status 'scheduled' e scheduled_at = data informada
E cria um job na fila BullMQ com delay calculado
E o bot responde: "✅ Agendado! Seu post será publicado em:
📅 Terça-feira, 09/04 às 18:00
📱 Plataformas: Instagram, TikTok, LinkedIn, YouTube

Para cancelar, acesse postai.app/dashboard ou responda 'cancelar agendamento {id}'"
```

**AC5 — Execução do job agendado**

```
Dado que chegou o horário agendado
Quando o job BullMQ é processado
Então o sistema publica o conteúdo em todas as plataformas selecionadas (mesmo fluxo da Story 2.1)
E atualiza campaign.status = 'published'
E envia notificação WhatsApp (ver Story 4.3)
```

**AC6 — Ver posts agendados**

```
Dado que o usuário tem posts agendados
Quando digita "agendados" ou acessa o menu de agendamento
Então o bot lista os próximos posts agendados:

"📅 Posts agendados:
1. 09/04 18h — Instagram, TikTok ✅
2. 12/04 09h — LinkedIn, YouTube ✅

Para cancelar: 'cancelar 1' ou 'cancelar 2'"
```

**AC7 — Cancelar agendamento via WhatsApp**

```
Dado que o usuário tem posts agendados listados
Quando digita "cancelar 1"
Então o bot confirma: "Cancelar o post do dia 09/04 18h? (sim/não)"
Se confirmado: remove o job da fila + atualiza status para 'cancelled'
E responde: "✅ Agendamento cancelado."
```

**AC8 — Agendamento para cliente (agência)**

```
Dado que Marina tem o cliente "Clínica Beleza Total" selecionado
Quando Marina agenda um post
Então o agendamento é criado com client_id da clínica
E ao executar, usa os tokens da clínica
```

---

## Notas Técnicas

- Parser de datas: usar `chrono-node` para linguagem natural em português
- Timezone: sempre armazenar em UTC, exibir no fuso do usuário (BRT por padrão = UTC-3)
- Job queue: BullMQ com Redis — job ID = campaign.id para permitir cancelamento
- Retry: mesmo padrão da Story 2.1 (3x com backoff) aplicado no job
- Agendamento máximo: 90 dias à frente
