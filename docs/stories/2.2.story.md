## User Story

Como **gestora de agência**, quero que o conteúdo publicado use as contas do cliente selecionado, para nunca misturar conteúdo entre clientes diferentes.

---

## Acceptance Criteria

**AC1 — Publicação sempre no contexto do cliente ativo**

```
Dado que Marina selecionou o cliente "Clínica Beleza Total" (client_id: abc123)
Quando Marina envia copy + vídeo e confirma publicação
Então o sistema busca api_tokens WHERE client_id = 'abc123'
E NÃO usa os api_tokens da conta de Marina
E publica nas contas da Clínica Beleza Total
```

**AC2 — Confirmação exibe nome do cliente**

```
Dado que Marina está no fluxo de publicação com cliente ativo
Quando o bot exibe o resumo para confirmação
Então o resumo inclui: "👤 Publicando como: Clínica Beleza Total"
E lista as plataformas conectadas desse cliente
```

**AC3 — Campanha registrada com client_id**

```
Dado que a publicação foi concluída
Quando o sistema salva a campanha
Então a campanha tem client_id = 'abc123' e agency_user_id = Marina_user_id
E o histórico de Marina mostra publicações separadas por cliente
```

**AC4 — Cliente sem plataforma conectada**

```
Dado que o cliente selecionado não tem nenhuma plataforma conectada
Quando Marina inicia o fluxo de publicação
Então o bot responde: "⚠️ Clínica Beleza Total não tem redes sociais conectadas ainda.
Conecte as redes em: postai.app/agency/clients/abc123/social"
```
