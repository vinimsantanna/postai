## User Story

Como **novo usuário**, quero assinar um plano do PostAI com cartão de crédito, para ativar minha conta e ter acesso ao bot de publicação.

---

## Acceptance Criteria

**AC1 — Página de planos**

```
Dado que o visitante acessa postai.app/pricing
Quando a página carrega
Então exibe 3 planos com preço, features e botão de assinar:

Creator  R$49/mês  → 1 usuário, 4 plataformas, ilimitado posts
Business R$149/mês → 1 usuário, 4 plataformas + agendamento + analytics
Agency   R$399/mês → até 10 clientes, multi-account, relatórios por cliente
```

**AC2 — Checkout via Stripe**

```
Dado que o usuário clicou em "Assinar Creator"
Quando é redirecionado para o Stripe Checkout
E preenche os dados do cartão
Então o Stripe processa o pagamento
E redireciona para postai.app/dashboard com status 'active'
```

**AC3 — Webhook de confirmação**

```
Dado que o Stripe processa pagamento com sucesso
Quando envia evento 'checkout.session.completed' para o webhook
Então o backend atualiza users.plan = 'creator' e users.status = 'active'
E envia email de boas-vindas com instruções do WhatsApp
```

**AC4 — Falha no pagamento**

```
Dado que o cartão é recusado pelo Stripe
Quando Stripe retorna falha
Então o usuário é redirecionado para postai.app/cadastro?error=payment_failed
E vê mensagem: "Pagamento não aprovado. Tente outro cartão."
E a conta NÃO é ativada
```

**AC5 — Renovação automática**

```
Dado que a assinatura está ativa
Quando o Stripe tenta cobrar na data de renovação
Se aprovado: conta continua ativa
Se recusado após 3 tentativas: status muda para 'past_due'
E bot responde: "⚠️ Seu pagamento falhou. Atualize seu cartão: postai.app/settings/billing"
```

---

## Notas Técnicas

- Webhook Stripe deve ser validado com `stripe.webhooks.constructEvent` (verificar signature)
- Planos como Stripe Products/Prices (não hardcoded)
- Guardar `stripe_customer_id` e `stripe_subscription_id` na tabela `users`

---

## Dependências

- Conta Stripe configurada com Products/Prices criados
