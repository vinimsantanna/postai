## User Story

Como **usuário do PostAI**, quero me identificar pelo meu CPF ao enviar a primeira mensagem no WhatsApp, para que o bot carregue meu perfil e contas conectadas sem precisar fazer login em um app.

---

## Acceptance Criteria

**AC1 — Sessão nova, usuário desconhecido**

```
Dado que o número de telefone do usuário não tem sessão ativa
Quando o usuário envia qualquer mensagem para o número do PostAI
Então o bot responde: "👋 Bem-vindo ao PostAI! Para começar, me informa seu CPF (só números):"
E NÃO executa nenhuma ação de publicação
```

**AC2 — CPF válido e cadastrado**

```
Dado que o bot está aguardando o CPF
Quando o usuário envia exatamente 11 dígitos numéricos (com ou sem máscara)
E o CPF existe na tabela users
Então o bot cria uma sessão ativa (whatsapp_sessions) vinculada ao user_id
E carrega nome, plano, api_tokens do usuário
E exibe o menu principal com saudação personalizada: "Oi {nome}! O que vamos fazer hoje?"
```

**AC3 — CPF válido mas não cadastrado**

```
Dado que o usuário enviou um CPF válido (11 dígitos)
Quando o CPF NÃO existe no banco
Então o bot responde: "❌ CPF não encontrado. Ainda não tem conta? Cadastre-se: postai.app/cadastro"
E NÃO cria sessão
```

**AC4 — CPF inválido (dígito verificador errado ou formato incorreto)**

```
Dado que o usuário enviou algo que não é um CPF válido
Quando o backend executa a validação de CPF
Então o bot responde: "⚠️ CPF inválido. Por favor, envie apenas os 11 números do CPF."
E mantém o estado aguardando CPF
```

**AC5 — Sessão já ativa (retorno)**

```
Dado que o usuário tem uma sessão ativa registrada para seu número de telefone
Quando o usuário envia qualquer mensagem
Então o bot NÃO solicita CPF novamente
E carrega a sessão existente diretamente
```

---

## Notas Técnicas

- CPF deve ser validado com algoritmo de dígito verificador (não apenas length=11)
- CPF armazenado sem máscara (`73988548309`) — remover pontuação/traço na entrada
- Sessão ativa = `whatsapp_sessions.status = 'active'` e `last_message` < 24h
- Timeout de sessão: 24h sem mensagem → sessão encerrada automaticamente
- Agências: após identificação, fluxo de seleção de cliente (ver Story 1.4)

## Dependências

- Schema `whatsapp_sessions`
