## User Story

Como **usuário identificado**, quero ver um menu claro após me identificar, para saber quais ações posso realizar e navegar facilmente pelo bot.

---

## Acceptance Criteria

**AC1 — Exibição do menu principal**

```
Dado que o usuário está identificado com sessão ativa
Quando o usuário envia qualquer mensagem fora de um fluxo específico (estado = 'menu')
Então o bot exibe:

"O que você quer fazer hoje, {nome}?

1️⃣ Publicar conteúdo agora
2️⃣ Agendar publicação
3️⃣ Ver histórico
4️⃣ Configurações
0️⃣ Ajuda"
```

**AC2 — Navegação por número**

```
Dado que o menu principal está sendo exibido
Quando o usuário digita "1"
Então o estado muda para 'waiting_for_copy'
E o bot responde: "📝 Qual é a copy do seu post? (texto da legenda)"

Quando o usuário digita "3"
Então o estado muda para 'history'
E o bot exibe os últimos 5 posts (ver Story 5.1)

Quando o usuário digita algo não reconhecido
Então o bot re-exibe o menu com: "Não entendi. Escolha uma opção:"
```

**AC3 — Persistência de estado entre mensagens**

```
Dado que o usuário iniciou um fluxo (ex: 'waiting_for_copy')
Quando o usuário envia a próxima mensagem
Então o estado da sessão é lido do banco antes de processar
E a resposta é adequada ao estado atual (não ao menu)
```

**AC4 — Comando de saída/cancelamento**

```
Dado que o usuário está em qualquer estado do fluxo
Quando o usuário digita "cancelar", "sair" ou "0"
Então o estado retorna para 'menu'
E o bot responde: "Ok! Voltando ao menu principal." + exibe menu
```

---

## Notas Técnicas

- State machine com estados: `menu`, `waiting_for_copy`, `waiting_for_video`, `waiting_for_thumbnail`, `confirm_publish`, `waiting_for_schedule`, `history`, `settings`
- Estado persiste em `whatsapp_sessions.current_step`
- Rascunho de campanha persiste em `whatsapp_sessions.campaign_draft` (JSONB)
- Recomendação: usar XState para formalizar as transições
